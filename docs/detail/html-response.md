# HtmlPageResponse: HTML-ответы в Joke

В микро-фреймворке **Joke** подход к генерации HTML имеет свои особенности. Класс `HtmlPageResponse`
осуществляет подход с HTML: страница собирается из компонентов (заголовков, стилей, скриптов и контента)
непосредственно перед отправкой клиенту.

## Текущий статус и планы

На данный момент основным классом для HTML-ответов остается базовый `HtmlResponse`. Он прост и легок, но требует от
разработчика самостоятельной заботы о структуре документа.

Однако **в следующей мажорной версии фреймворка ситуация изменится**:

* **`HtmlPageResponse` станет основным классом** по умолчанию для всех HTML-страниц.
* **`HtmlResponse` перейдет в разряд вспомогательных**, предназначенных для специфических случаев (например, отдачи
  полностью готового статического кэша).

Такой переход обусловлен тем, что `HtmlPageResponse` предоставляет архитектурные преимущества, необходимые для
современных приложений: централизованное управление ресурсами, безопасность и гибкость модификации ответа мидлварами.

## Ключевые возможности

### 1. Доступ к конструктору страницы (`builder`)

Свойство `$builder` доступно для чтения (`public private(set)`). Это дает вам полный контроль над структурой документа
прямо из контроллера или сервиса:

```php
$response = new HtmlPageResponse($container);

// Управление мета-информацией
$response->builder->setTitle('Моя страница')
                  ->addMeta('description', 'Описание');

// Управление атрибутами тегов
$response->builder->htmlAttributes->set('lang', 'ru');
$response->builder->bodyAttributes->append('class', 'dark-theme');
```

### 2. Интеллектуальный `setBody`

Метод умеет различать фрагменты и полные документы:

* **Фрагмент:** Становится содержимым `<body>`.
* **Полный документ:** Разбирается через `HtmlImporter` (на базе `DOMDocument`). Заголовки, мета-теги и атрибуты
  извлекаются и интегрируются в билдер.

```php
// Импорт legacy-страницы с сохранением контроля
$response->setBody($fullLegacyPageHtml);
$response->builder->addMeta('robots', 'noindex'); // Добавляем поверх импорта
```

### 3. Управление ресурсами (Assets)

Вы можете подключать файлы из любого места кода, не боясь дубликатов:

```php
// Автоматическая дедупликация и версионирование
$response->builder->css->addToHead('/css/main.css');
$response->builder->js->addToBody('/js/app.js');
```

**AssetFileManager** автоматически добавляет хэш версии файла к URL, решая проблему кэширования браузером.

## Архитектура под капотом

`HtmlPageResponse` — это фасад над системой компонентов:

| Компонент            | Назначение                                                     |
|:---------------------|:---------------------------------------------------------------|
| **PageBuilder**      | Генерирует финальный HTML из частей.                           |
| **HtmlImporter**     | Парсит сырой HTML и переносит его структуру в билдер.          |
| **AssetCollection**  | Управляет списками файлов, их порядком и позицией (Head/Body). |
| **StringCollection** | Хранит уникальные блоки кода (инлайн-скрипты, стили).          |

## Пример использования

```php
class PageController 
{
    public function index(ServiceContainer $container): HtmlPageResponse
    {
        // Инициализация с передачей контейнера
        $response = new HtmlPageResponse($container);
        
        // Настройка страницы через публичный билдер
        $response->builder->setTitle('Главная')
                          ->addMeta('viewport', 'width=device-width');
                
        // Подключение ресурсов
        $response->builder->css->addToHead('/assets/style.css');
        
        // Установка контента
        $response->setBody($this->view->render('index'));
        
        return $response;
    }
}
```

## Заключение

Сейчас можно использовать `HtmlPageResponse` для задач, где требуется тонкий контроль над структурой страницы. А в
следующей мажорной версии он станет основным для HTML-слоя фреймворка. Рекомендую уже сейчас начинать знакомство с этим
классом, чтобы будущий переход прошел максимально гладко.
