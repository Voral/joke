# Маршрутизация

Joke предоставляет гибкую систему маршрутизации, основанную на явной регистрации обработчиков для HTTP-методов и
URI-паттернов. Все маршруты определяются в конфигурационных файлах и тесно интегрированы с DI-контейнером и слоем
middleware.

## Конфигурация маршрутов

По умолчанию настройка маршрутов производится в файле `routes/web.php`. Путь к файлу маршрутов можно переопределить в
конфигурационном файле приложения `config/app.php`:

```php
<?php

return [
    'fileRoutes' => 'routes/my-app.php',
];
```

> **ВАЖНО!**  
> Web-маршруты автоматически оборачиваются в стандартный стек middleware:
> * Управление сессией (по умолчанию в блокирующем режиме).
> * Защита от CSRF-атак.

## Базовое использование

Внутри файла маршрутов доступна переменная `$router` типа `Vasoft\Joke\Routing\Router`. Простейший маршрут
регистрируется так:

```php
use Vasoft\Joke\Routing\Router;

/** @var Router $router */
$router->get('/hello', fn() => 'Hello World');
```

Обработчик может быть любым вызываемым значением: замыканием, функцией, методом класса или invokable-объектом.

## Поддерживаемые HTTP-методы

Маршрутизатор поддерживает все основные HTTP-методы:

```php
$router->get($path, $handler);    // Получение данных (идемпотентен)
$router->head($path, $handler);   // Получение только заголовков (идемпотентен)
$router->post($path, $handler);   // Создание ресурса
$router->put($path, $handler);    // Полная замена ресурса (идемпотентен)
$router->patch($path, $handler);  // Частичное обновление ресурса
$router->delete($path, $handler); // Удаление ресурса (идемпотентен)
$router->options($path, $handler);// Запрос возможностей (идемпотентен)
```

Если маршрут должен отвечать на любой метод запроса, используйте специальный метод:

```php
$router->any($path, $handler);
```

Также возможна регистрация маршрута на строго заданный перечень HTTP-методов:

```php
use Vasoft\Joke\Http\HttpMethod;

$router->match([HttpMethod::GET, HttpMethod::POST], $path, $handler);
```

### Автоматическая обработка OPTIONS

Фреймворк имеет встроенную поддержку метода `OPTIONS`. Если для запрашиваемого URI не найден явный маршрут, но метод
запроса — `OPTIONS`, роутер автоматически вернет ответ со статусом `200 OK` и заголовком `Allow`, содержащим
список всех поддерживаемых методов. Это необходимо для корректной работы CORS (Preflight-запросы) без написания
дополнительного кода.

## Внедрение зависимостей (DI)

Фреймворк автоматически разрешает зависимости через DI-контейнер. Вы можете объявлять в сигнатуре обработчика любые
типизированные параметры, зарегистрированные в контейнере:

```php
use Vasoft\Joke\Routing\Router;
use Vasoft\Joke\Http\HttpRequest;

/** @var Router $router */
$router->get('/hello', function(HttpRequest $request) {
    // $request будет автоматически внедрен контейнером
    return 'Hello, ' . $request->ip;
});
```

## Типы обработчиков

Поддерживаются следующие формы записи обработчиков:

```php
// Замыкание
$router->get('/a', fn() => 'hi');

// Статический метод через строку (Class::method)
$router->get('/b', 'SomeClass::staticMethod');

// Callable в стиле first-class (PHP 8.1+)
$router->get('/c', SomeClass::staticMethod(...));

// Массив [класс, метод]
$router->get('/d', [SomeClass::class, 'staticMethod']);

// Массив [объект, метод]
$instance = new SomeClass();
$router->get('/f', [$instance, 'staticMethod']);

// Invokable-класс (экземпляр или имя класса)
$router->get('/g', InvokeController::class);
$router->get('/h', new InvokeController());
```

## Параметры маршрутов

### Базовые параметры

Сегменты URI могут быть захвачены как параметры с помощью синтаксиса `{имя}`:

```php
$router->get('/order/{userId}/{id}', function(int $userId, int $id) {
    // ...
});
```

Параметры автоматически передаются в обработчик в соответствии с именами и типами аргументов.

> Важно: Параметры маршрута передаются в обработчик "как есть". Если вы используете их для построения файловых путей или
> SQL-запросов, вы обязаны самостоятельно валидировать и санитизировать эти данные. Не доверяйте входным данным из URL.

### Автоматическая десериализация

Если тип аргумента поддерживает преобразование из строки (например, `enum` или классы с методом `tryFrom`), фреймворк
попытается автоматически преобразовать значение:

```php
$router->get('/order/{status}', function(OrderStatus $status) {
    // $status будет объектом OrderStatus
});
```

### Ограничение параметров

Можно задать шаблон проверки для параметра через двоеточие: `{имя:правило}`:

```php
$router->get('/catalog/{section:slug}/{id:int}', function(string $section, int $id) {
    // ...
});
```

Доступные правила:

* `:int` → соответствует `\d+`
* `:slug` → соответствует `[a-z0-9\-_]+`
* По умолчанию → соответствует `[^/]+` (любой символ кроме слэша)

### Wildcard-параметр (`{*}`)

Для перехвата **всего оставшегося пути** (включая вложенные слэши) используется специальный параметр `{*}`:

```php
$router->get('/{*}', fn() => 'Страница не найдена');
```

Этот паттерн:

* Совпадает с **любым URI**, например `/user/123/profile` или `/api/v1/data.json`.
* Должен регистрироваться **в конце списка маршрутов**, чтобы не перекрывать более конкретные правила.
* Автоматически передаётся в обработчик под именем `$path`:

```php
$router->get('/{*}', function(string $path) {
    return "Запрошен несуществующий путь: {$path}";
});
```

> Wildcard-маршрут особенно полезен для реализации страницы «404 Not Found».

## Именованные маршруты

Каждому маршруту можно присвоить уникальное имя для последующего обращения или генерации URL:

```php
// Третий аргумент - имя маршрута
$router->get('/profile', fn() => '...', 'profile');

// Получение объекта маршрута по имени
$route = $router->route('profile');
```

## Группировка маршрутов

Маршруты можно объединять в группы, чтобы применять к ним общие настройки или middleware:

```php
$router->get('/hello', fn() => 'hi', 'hello')
    ->addGroup('filtered')
    ->addGroup('second');
```

Маршруты, подключаемые из файла веб-маршрутов (`routes/web.php`), автоматически добавляются в группу `web`.
Фреймворк включает enum со стандартными группами: `Vasoft\Joke\Routing\StdGroup`.

## Middleware на уровне маршрута

Для каждого маршрута можно назначить дополнительные middleware:

```php
$router->get('/admin', fn() => 'Admin Panel', 'admin')
    ->addMiddleware(AuthMiddleware::class)
    ->addMiddleware(LogMiddleware::class, 'logger');
```

Middleware выполняются до вызова основного обработчика и могут модифицировать запрос, ответ или прерывать выполнение.

> **ВНИМАНИЕ!**  
> Именованные посредники уровня маршрута могут переопределять посредников с тем же именем, зарегистрированных на уровне
> роутера глобально. Они будут выполнены в той позиции очереди, которая была определена при глобальной регистрации
> имени.

## Расширение через контракты

Фреймворк предоставляет интерфейсы для полной замены компонентов маршрутизации:

* `Vasoft\Joke\Contract\Routing\RouteInterface`
* `Vasoft\Joke\Contract\Routing\RouterInterface`

Чтобы использовать собственную реализацию, зарегистрируйте её в DI-контейнере до запуска приложения:

```php
// bootstrap/app.php

use Vasoft\Joke\Contract\Routing\RouterInterface;
use Vasoft\Joke\Application\Application;
use Vasoft\Joke\Container\ServiceContainer;
use App\MyCustomRouter;

$container = new ServiceContainer();
// Регистрируем свою реализацию вместо стандартной
$container->registerSingleton(RouterInterface::class, MyCustomRouter::class);

return new Application(dirname(__DIR__), '', $container);
```

Это позволяет кастомизировать поведение маршрутизатора (например, изменить алгоритм поиска или формат паттернов) без
изменения ядра фреймворка.
