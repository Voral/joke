# Структура каталогов

Joke не навязывает строгую архитектуру, но рекомендует следующую организацию проекта. В будущем эта структура будет
автоматически генерироваться с помощью скелетона.
```text
/  
├── bootstrap/   # Инициализация приложения  
├── public/      # Публичная точка входа и статические ресурсы  
├── routes/      # Определение маршрутов  
└── src/         # Исходный код приложения
```

## Каталог bootstrap

Содержит файл app.php, в котором выполняется первоначальная настройка приложения:

- создание DI-контейнера,
- регистрация сервисов,
- подключение маршрутов и middleware.

Этот файл подключается из точки входа (public/index.php) и возвращает готовый экземпляр приложения.

## Каталог public

Публичная директория — единственная, доступная напрямую извне.

- index.php — центральная точка входа для всех HTTP-запросов.
- Здесь же размещаются статические ресурсы: CSS, JavaScript, изображения и т.д

> Убедитесь, что веб-сервер настроен так, чтобы запретить прямой доступ ко всем остальным каталогам проекта.

## Каталог routes

Содержит файлы с определением маршрутов. На текущем этапе поддерживается только веб-маршрутизация (`web.php`).

При обработке этих маршрутов автоматически:

- запускается сессия,
- применяется CSRF-защита.

## Каталог src

Основной каталог исходного кода вашего приложения: контроллеры, сервисы, DTO, обработчики и другие классы.

Рекомендуется использовать PSR-4 автозагрузку и организовывать код по принципам разделения ответственности.
