# Планируемые изменения

## Изменения планируемые в следующей версии

- [ ] Реализовать кэширование результатов рефлексии для повышения производительности
- [ ] Добавить поддержку параметров со значениями по умолчанию
- [ ] В связывании параметров учесть юнион типы
- [ ] CORS
- [ ] Механизм работы с куками
- [ ] Конфигурируемая отправка CSRF токена через куки
- [ ] При определении https реализовать поддержку доверенных прокси для корректной работы за балансировщиками (заголовки
  типа X-Forwarded-Proto)

## Изменения планируемы в версии 2

- [ ] Добавление метода abstract public function getContentType(): string; в абстрактный класс Response. ContentType в
  заголовки будет добавляться в методе Response::send()
- [ ] Убрать пространство Core
- [ ] Минимальная версия PHP 8.5
- [ ] В DiContainer изменить поведение. Если сервис не найден бросать исключение (совместимость с PSR11)
- [ ] DiContainer изменить поведение register с учетом __invoke()
- [ ] DiContainer методы register* сбудут возвращать контейнер
- [ ] Повысить уровень проверки стана
- [ ] Для роута и групп роутов тип ответа по умолчанию и возможность его задать
- [ ] Параметр Secure для кук изменять если запрос по http (для локальной разработки). Изучить необходимость.
